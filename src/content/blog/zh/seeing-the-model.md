生成式系统把等待重新带回了界面。它不是传统意义上可以精确估计的等待，而是一段充满推断、修正和不确定性的时间。

## 进度条已经失效

传统进度条假设任务可以被切分，剩余时间可以被估算。模型生成并不完全符合这个前提。用一个匀速增长的数字掩盖不确定性，只会制造错误承诺。

我们需要呈现的不是“还剩多少”，而是**系统此刻正在处理什么层级的问题**。状态可以被组织成三个尺度：意图、材料和组合。

```ts
type GenerativeState = {
  intent: 'reading' | 'planning' | 'revising';
  confidence: number;
  canInterrupt: boolean;
};
```

## 从状态到空间

当状态具有空间关系，用户可以理解变化，而不需要阅读冗长日志。聚拢表示收敛，分叉表示存在候选路径，低频呼吸则意味着系统仍在工作但暂时不需要介入。

## 保留打断的权利

任何生成过程都应该允许暂停、改变方向和回到上一步。可解释性不只是告诉用户发生了什么，也包括让用户知道自己可以做什么。

好的等待界面不会假装确定。它把不确定性转化成可以感知和操作的材料。
